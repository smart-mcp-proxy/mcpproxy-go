# Contract: `mcpproxy-apt` R2 Bucket Layout

Custom domain: `apt.mcpproxy.app`. Public read.

## Object keys

```
apt.mcpproxy.app/
├── mcpproxy.gpg                                    # public key (binary OpenPGP)
├── mcpproxy.gpg.asc                                # public key (ASCII-armored; convenience)
├── dists/
│   └── stable/
│       ├── Release                                 # top-level suite manifest (signed by Release.gpg)
│       ├── Release.gpg                             # detached signature
│       ├── InRelease                               # clearsigned Release (preferred by modern apt)
│       └── main/
│           ├── binary-amd64/
│           │   ├── Packages                        # plaintext index
│           │   └── Packages.gz                     # gzip
│           └── binary-arm64/
│               ├── Packages
│               └── Packages.gz
└── pool/
    └── main/
        └── m/
            └── mcpproxy/
                ├── mcpproxy_{VERSION}_amd64.deb    # up to 10 amd64 versions
                └── mcpproxy_{VERSION}_arm64.deb    # up to 10 arm64 versions
```

## Naming conventions

- **Suite**: `stable`
- **Component**: `main`
- **Pool path**: `pool/main/m/mcpproxy/` — the `m/mcpproxy/` segment follows Debian's first-letter-of-source-package convention.

## `dists/stable/Release` expected fields

```
Origin: MCPProxy
Label: MCPProxy
Suite: stable
Codename: stable
Architectures: amd64 arm64
Components: main
Description: MCPProxy stable release channel
Date: <RFC 2822 UTC>
SHA256:
 <sha>  <size>  main/binary-amd64/Packages
 <sha>  <size>  main/binary-amd64/Packages.gz
 <sha>  <size>  main/binary-arm64/Packages
 <sha>  <size>  main/binary-arm64/Packages.gz
```

Generated by `apt-ftparchive release` using `apt-ftparchive.conf`.

## Cache headers (set on upload)

Uploaders set HTTP cache metadata via `aws s3 cp --metadata-directive REPLACE --cache-control "..."`:

| Path pattern | Cache-Control |
|---|---|
| `dists/stable/Release*` | `public, max-age=60, must-revalidate` — changes on every release |
| `dists/stable/InRelease` | `public, max-age=60, must-revalidate` |
| `dists/stable/**/Packages*` | `public, max-age=60, must-revalidate` |
| `pool/main/m/mcpproxy/*.deb` | `public, max-age=31536000, immutable` (forever) |
| `mcpproxy.gpg*` | `public, max-age=86400` (1 day) |

The 5-minute TTL on metadata bounds the CDN staleness window and satisfies the "staleness clears within 1 hour" edge case.

## User-facing install snippet

```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://apt.mcpproxy.app/mcpproxy.gpg \
  | sudo tee /etc/apt/keyrings/mcpproxy.gpg > /dev/null
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/mcpproxy.gpg] https://apt.mcpproxy.app stable main" \
  | sudo tee /etc/apt/sources.list.d/mcpproxy.list > /dev/null
sudo apt update
sudo apt install mcpproxy
```
